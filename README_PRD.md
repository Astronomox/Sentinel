# SENTINEL — Product Requirements Document
**Threat Intelligence Platform | Claude Hackathon 2026**

> Version: v1.0 | April 2026 | Classification: CONFIDENTIAL

---

## 1. Product Overview

SENTINEL is a real-time cybersecurity threat monitoring and intelligence platform powered by Claude AI. It ingests security event data, classifies threats by severity, surfaces non-obvious attack patterns, and dispatches structured alerts — all without requiring user authentication. The dataset is brought in by the operator.

### 1.1 Problem Statement

Security teams are overwhelmed by high-volume, low-signal alert noise. Existing tools lack the reasoning capacity to correlate threats in real time and generate human-readable intelligence reports. SENTINEL solves this by replacing rule-based alerting with Claude-powered contextual analysis.

### 1.2 Goals

- Provide a live, auto-updating feed of classified cyber threats
- Use Claude AI to reason over threats and generate actionable intelligence
- Enable one-click alert dispatch and structured report generation
- Support custom dataset ingestion without authentication overhead
- Deliver a production-grade dashboard suitable for demo and deployment

### 1.3 Non-Goals

- No user login, authentication, or role-based access control
- No persistent database — operator brings the dataset
- No mobile-native app (web-responsive only)
- No third-party SIEM integration in v1

## 3. Feature Requirements

### 3.1 Live Threat Feed
- Auto-ingests threat events from the operator-supplied dataset
- Classifies each event as CRITICAL / HIGH / MEDIUM / LOW
- Updates the feed every 6 seconds with new events
- Color-coded severity indicators and source IP display
- Clickable items that trigger Claude analysis on selection

### 3.2 Claude AI Analysis Engine
- On-demand threat analysis triggered by selection or manual button
- Sends threat metadata to Claude via Anthropic API
- Typewriter effect renders analysis in real time
- Output includes: classification, technical analysis, recommended actions
- Context-aware: uses threat name, severity, source, and timestamp

### 3.3 Statistics Dashboard
- Live counters for Critical, High, Medium, Low, and Total threats
- Delta indicators showing change from last hour
- Auto-increments as new threats arrive

### 3.4 Geographic Attack Map
- Visual map overlay showing active attack origins
- Animated ping dots representing live attack sources
- Color differentiation by severity (red = critical, amber = high)

### 3.5 Alert Dispatch
- One-click alert sends notification for selected/active threat
- Alert logged in right panel with timestamp and severity
- Visual flash animation on new alert
- Toast notification confirms dispatch
- Alert count tracked in session

### 3.6 Full Intelligence Report
- Claude generates a structured multi-section threat report
- Covers: Executive Summary, Threat Landscape, Top Threats, Attack Vectors, Risk Assessment, Recommended Actions, Incident Response Priority
- Rendered in modal overlay with scroll
- Export to `.txt` file with timestamp

### 3.7 Dataset Ingestion
- No authentication required — operator loads dataset directly
- Supports custom threat data in defined JSON schema
- Schema: `{ name, sev, sources[], timestamp }`
- Fallback to built-in mock data for demo purposes

## 4. Technical Specifications

| Component | Specification |
|---|---|
| Frontend | Vanilla HTML/CSS/JS — single file, no build step |
| AI Engine | Anthropic Claude API — `claude-sonnet-4-20250514` |
| Auth | None — no login required |
| Data Layer | Operator-supplied dataset (JSON) + in-memory state |
| Styling | CSS Variables, Grid layout, custom fonts (Orbitron, Rajdhani, Share Tech Mono) |
| Alerts | In-session log — no persistence |
| Reports | Generated by Claude, exported as `.txt` |
| Deployment | Static file — any web server or CDN |
